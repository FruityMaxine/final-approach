## FA组11 Tick3 (多发引擎模型 engines.js · repo v0.7.0.0)
**改了什么**：新增 web/engines.js 多发引擎模型,把"单一合集推力"重构为可配置多发(2/4发)独立模型:
- 每发独立 {n1,n2,egt,ff,state,throttleOvr,fuelCut,fire,starter,ign},状态机 off/start/idle/run/fail/fire,engineStep(dt) 推进各发 spool(N1惯性)/N2/EGT(随N1)/FF燃油流量;启动序列(starter→N2升→点火→idle→run)。
- 物理总推力 T = ENGINES.totalThrust() = Σ各发(失效发动机贡献≈0,总推力自然下降);反推地面取负。
- **不对称推力偏航**:ENGINES.yawFromThrust() 按各发横向力臂×推力求矩,接入 game.js 偏航 psiDot(非装饰)——单发失效把机头拉向死发一侧。
- EICAS 动态显示:renderEICAS() 按发动机数生成 2 或 4 列 N1 条+数值+EGT,失效列变红。
- 故障注入接口 failEngine/fireEngine/cutoff(供 Tick4 系统面板+组12故障系统调用)。
- 配置:发动机数(双发/四发)入飞行设置面板+CONFIG.engines+localStorage持久化;S.N1 取队均供 UI/EMMA/声音向后兼容。
**为什么改**：用户要硬核多发引擎独立控制。这是引擎/燃油/系统面板(Tick4-5)与故障系统(组12)的物理基础。
**影响范围**：新增 engines.js(~95行);game.js(物理 spool/推力/偏航改 + resetState引擎重置 + renderEICAS + syncRuntimeUI多发显示 + init setCount + 配置seg,~60行);config.js(CONFIG.engines+持久化);index.html(engines.js script + 发动机seg);game.css(.failed/.egtnum);game.json(modules+0.6→0.7.0.0)。
**实测**：Playwright — 双发/四发EICAS动态列(2/4 units);单发失效航向0°→13.24°偏向死发(yaw moment 0.169,确认偏航接物理非装饰);E1失效退转state=fail/n1=0.05;同油门总推力=Σ下降;零console错误;截图确认四发EICAS。

## FA组11 Tick4 (面板框架 PanelFramework + 引擎系统面板 · repo v0.8.0.0)
**改了什么**：新增 web/panels.js 面板框架 —— 不写死单面板,而是可注册数十种驾驶舱面板的框架:
- PANELS.register(id,{label,build,wire,sync}) 注册即生效,选择器 tab 条自动出条目;open(id)显示覆盖层,close()回飞行视图;SYS.panels 开关过滤面板可见性。
- 面板选择器 #panelBar(飞行 + 已注册面板,如 引擎)底部 tab 条;#panelHost 覆盖层覆于世界视图上(飞行继续),每帧 PANELS.sync() 刷新当前面板读数。
- 首个注册面板=引擎/燃油系统面板:每发一列(ENG 1..N),显示 state+N1/N2/EGT/FF,5 按钮(起动机/点火IGN/断油CUTOFF/灭火手柄/燃油泵)。
- **按钮真连动 engines.js 状态机**:CUTOFF→fuelCut停车、STARTER+IGN→启动序列(N2起转点火至慢车run)、灭火手柄→断油灭火,非纯显示。
**为什么改**：用户北极星"真飞机几十种面板都能做出来"——建框架而非一次性面板,引擎面板只是首个,MCDU/液压/电气等后续按框架无限加。
**影响范围**：新增 panels.js(~95行);engines.js(加pump字段);index.html(#panelBar/#panelHost+panels.js script);game.css(面板框架+引擎列+按钮样式~50行);game.js(PANELS.init+loop sync);game.json(modules+0.7→0.8.0.0)。
**实测**：Playwright — 选择器 tabs[飞行,引擎];开引擎面板 panelHost.show+2列+current=engine;CUTOFF eng1→state=off/n1=0.00(按钮连动状态机非装饰);重启序列(CUTOFF关+STARTER+IGN)→state=run/n2=0.63;零console错误;截图确认引擎面板硬核视觉。

## FA组11 Tick5 (燃油系统 fuel.js · repo v0.9.0.0 · 组11收官)
**改了什么**：新增 web/fuel.js 燃油系统:
- TANK{left,center,right}各{qty,cap}+pump泵+xfeed交输活门;FUEL.step(dt)按各发FF(engines.js)扣油,路由:中央先用→同侧机翼箱,crossfeed开则跨侧供油,泵关的箱不供油。
- **燃油饥饿真连锁**:某发供油不足(箱空+无crossfeed)→e.fuelStarve+state转fail(engines状态机熄火),非数字显示。
- 油量影响总重 AC.m=dryMass(44000)+总油量(满油18000→62000不变,烧油变轻)。
- 新增"燃油"面板(PANELS第2个注册面板,验证框架可扩):三油箱竖条synoptic+泵开关+交输活门+总量。
- SYS登记features.fuelSystem(关=无限油简化档)+panels.fuel。
**为什么改**：用户硬核燃油系统诉求(左中右油箱/泵/交输/耗油)。燃油饥饿熄火是引擎×燃油跨模块连锁,也为组12故障系统(燃油泄漏)铺路。
**影响范围**：新增fuel.js(~90行);game.js(AC.dryMass+FUEL.step/reset);engines.js(启动清fuelStarve);sys.js(fuelSystem+fuel面板登记);index.html(fuel.js script);game.css(油箱/交输样式~22行);game.json(modules+0.8→0.9.0.0)。
**实测**：Playwright — 面板tabs[飞行,引擎,燃油](框架第2面板);耗油递减;右+中空+交输关→eng2 fail+fuelStarve真熄火、eng1仍run(跨模块连锁);右+中空+交输开→eng2跨供left箱仍run;synoptic三油箱;零console错误;截图确认燃油面板。
